오픈소스아카데미 WEB 과정 HTML+CSS(수코딩)

1. 프론트엔드와 백엔드의 개념
 "백엔드를 하려면 그래도 프론트엔드는 배워라"
 
2. 태그
	콘텐츠가 있는 태그(</>태그로 닫을 수 있음)/없는 태그(빈 태그)
	
	태그의 속성: 태그를 의미나 기능적으로 보조
	ex) lang="en": <html>태그를 의미나 기능적으로 보조
	
3. 기본 구조
	-<!DOCTYPE html>: 최신 버전의 html 사용!
	-<html>: html의 시작을 알림
	-<head>: 문서의 정보 표현(브라우저에 표시 안됨)
		-<meta>: 인코딩과 뷰포트 등을 결정
		-<title>: 문서의 제목(탭 이름으로 표기)
	-<body>: 브라우저에 표현되는 내용
	
4. h(n) 태그
	헤딩 태그, 콘텐츠의 대제목, 중제목, 소제목
	h1~h6 순으로 글자가 작아짐
	
	*검색 엔진 최적화를 위한 원칙
	 1. h1은 문서 전체에서 한 번만
	 2. 1부터 순차적으로(건너뛰면 안 됨)

5. p 태그
	paragraph, 내용적으로 연결된 하나의 문단
	외형적인 스타일링 없이 출력, 문단 사이에는 한 줄이 떨어짐
	
	가로 한 줄을 모두 사용, 창의 크기에 따라 유동
	모든 공백은 한 칸으로 표시되므로, &nbsp;(스페이스)와 <br>(줄바꿈)을 이용해서 띄움

6. a 태그
	anchor, 링크
	
	-내부 링크
		href="#one"가 있는 a태그를 클릭하면
		id="one"가 있는 p태그로 이동
		(나무위키의 목차 등으로 사용)
		
	-외부 링크
		href="URL"
		target="_blank" 속성을 추가하면 새 탭에서 링크 열기 가능

7. img 태그
	<img src="경로">
	-alt 속성: 잘못된 경로로 이미지가 깨졌을 때 대체 텍스트 출력
	-heighth/width 속성: 가로/세로(픽셀 단위나 퍼센트)
		단독으로 쓰였을 때는 나머지 하나가 자동으로 맞춰지지만, 둘다 쓰면 비율이 손상될 수 있음
	
	-경로로 외부 리소스 이미지 사용 가능
	-내부 경로: /로 폴더구분, ./: 현재 폴더, ../: 상위 폴더
